### IP 주소를 사용하는 이유

- 세상에 있는 모든 MAC 주소를 알기 어렵기 떄문
- MAC 주소를 알아야 통신이 가능함

IP주소는 알지만 MAC 주소는 모르는 경우가 더 많다.

ARP
: IP 주소("동일" 네트워크 내의 호스트)를 통해 MAC 주소를 알아내기 위한 프로토콜

ARP 동작과정
0. IP 주소만 아는 상태(MAC 주소는 모름)

1. ARP 요청이라는 패킷을 먼저 보냄
2. ARP 응답을 받는다.
3. ARP 테이블(ARP 캐시) 갱신
4. IP와 MAC 주소 모두 아는 상태

ARP 요청

- 특정 주소를 가진 호스트의 MAC 주소를 알아내기 위해 보내는 브로드 캐스트 메시지
- 해당 호스트의 MAC 주소를 모르기 때문에 브로드 캐스트 메시지로 전송
브로드캐스트 메시지는 많이 보내면 트래픽 발생

ARP 응답

- ARP 요청 메시지에 대한 응답. 자신의 MAC 주소 포함.
- ARP 패킷에서 오퍼레이션 코드에 따라서 응답인지 요청인지 구분함.
